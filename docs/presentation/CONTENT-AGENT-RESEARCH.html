Content Agent Research Tools

Post-session technical content analysis now grounds scores in live reference lookup — official docs via Context7 and articles/improvement guidance via Exa — before the content LLM evaluates the transcript.

Branch: content-agent Spec: specs/001-content-agent-research-tools/ 39 tasks completed 13 tests passing

Summary

Before

Content agent read transcript + topic only. LLM relied on static training data. No external research.

After

Technical topics trigger classify → parallel Context7 + Exa fetch → inject reference blocks into evaluation prompt → scored findings with research_status.

Resilience

DEMO_MODE=replay bypasses all research. Missing keys or timeouts degrade to transcript-only with explicit UI/API status — report never blocks.

Pipeline

POST /sessions/:id/end → ReportAgent.generate() → ContentAnalysisAgent.analyse()

1. Load session, transcript, slide analyses from DB → replay? fixtures & exit
2. classify_topic(topic, topic_context) // LiteLLM structured output
3. if technical: fetch_reference_bundle() // parallel, 20s budget ├─ Context7: resolve library + query docs └─ Exa: search_articles + search_improvements
4. _build_evaluation_prompt(docs + articles + improvements + slides + transcript)
5. Agno Agent + ContentResult schema → ContentAnalysisPayload
→ ReportAgent scores 4 vectors, persists report + content_research_status
→ GET /sessions/:id/report + ScoreCard UI disclaimer

Exa integration

Python SDK (exa-py, AsyncExa). Two fixed query templates per session topic:

FunctionQuery templatePurpose
search_articles() "{topic}" official features documentation overview Coverage + factual grounding from expert articles
search_improvements() "{topic}" best practices common mistakes what to cover technical talk Actionable improvement insights (pitfalls, audience expectations) 
await exa.search(query, num_results=5, contents={"highlights": True})

Context7 integration

HTTP via httpx (no MCP subprocess). Uses classifier primary_libraries (max 2).

EndpointPurpose
GET /api/v2/libs/search Resolve library name → ID (e.g. /facebook/react)
GET /api/v2/context Fetch doc snippets for topic query (type=json)

Official docs take precedence over Exa articles for factual disputes in the content LLM prompt.

research_status

full All configured providers returned snippets
partial Some providers succeeded; insight prepended to report
skipped Technical topic but no research (keys missing / all failed)
not_applicable Non-technical topic, empty transcript, or replay mode

How to test

Automated

make db-migrate
cd packages/backend && uv run pytest tests/test_content_research.py tests/test_report_dedup.py -q

Live (UI)

  • make dev — ensure DEMO_MODE is empty
  • Start session with technical topic (e.g. "React 19 new features")
  • Rehearse with mic so transcript is captured
  • End session → check Technical Content panel + footer disclaimer
  • Look for content_research_status: "full" or "partial" in report JSON

Safe demo

  • Set DEMO_MODE=replay — no Exa/Context7 calls, fixture content findings

Degradation

  • Clear EXA_API_KEY and CONTEXT7_API_KEY → expect skipped + fallback insight

Constitution alignment

  • Demo path preserved — replay mode unchanged; upload → rehearse → end → report
  • Agent boundaries — research is read-only; ReportAgent still owns report writes
  • Resilience — bounded timeouts, partial fallback, no report failure on provider errors
  • Stack simplicity — one new dep (exa-py); Context7 via existing httpx
