Why

Problem & Solution

Problem

  • Engineers prep talks with no feedback loop before going on stage
  • Speaking coaches expensive + asynchronous
  • Self-recorded video = tedious, unstructured, no metrics
  • Vague impressions, no quantified path to improve

Solution

  • Always-on AI co-pilot watches + listens in real time
  • Upload → Record → Analyse → Report pipeline
  • 4-vector scorecard: voice · body · slides · content
  • Timestamp + slide anchored insights · score /100 · mentor unlock ≥80

Golden rule: agents are pure AI logic — they emit typed Pydantic payloads to the Orchestrator and never touch the DB. Orchestrator is the sole writer; Report + Content agents get read-only access for aggregation.

Algorithm · cost control

Frame Delta Filtering

flowchart TB
    A([📷 getUserMedia
60fps capture]) --> B["downscale 320×240
JPEG q0.7 · every 200ms"]
    B --> C{{"dhash
Hamming distance"}}
    C -->|"≤ 8 · duplicate"| D["🗑️ drop frame"]
    C -->|"> 8 · changed"| E["queue for batch"]
    E --> F["batch 3 frames
@ ~600ms"]
    F --> G[("🤖 GPT-4o Vision
1 call / batch")]
    class A sky
    class B,E,F amber
    class C violet
    class D red
    class G green
  • 60fps would be ruinous on latency + token cost — banned
  • imagehash dhash → Hamming distance vs last frame
  • Distance ≤ 8 → frame is redundant → drop silently
  • Survivors batched 3 per call to amortise tokens (~1000 calls / 10-min session)
  • Result: ≤5fps reaches the Vision Agent

Two UUIDs, zero login. session_id public (safe in URLs/logs), access_token secret in localStorage. Both required for every protected op. WS can't send headers → ?token= query param, validated on handshake, close(4003) if invalid. Opt-in share_token grants read-only report access.

Rationale

Why These Choices

Single FastAPI process

  • All agents = asyncio tasks, zero inter-service networking
  • Right for a 6-hour build · risk: blocking call stalls loop → run_in_executor

In-process pub/sub

  • asyncio.Queue + dict[session_id → set[WS]]
  • No Redis · clean interface, swappable post-hackathon

Orchestrator = sole DB writer

  • Agents pure AI, emit typed payloads · Pydantic validation wall
  • VideoEvents batched 1s/N=20 to avoid write bottleneck

Capability tokens

  • Full session isolation, no login flow
  • Agno is brand-tax — fallback: raw asyncio.create_task
